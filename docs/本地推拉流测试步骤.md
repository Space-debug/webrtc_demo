# 本地 UDP 推拉流完整测试步骤

按以下步骤操作，可复现成功的推拉流。

---

## 前置条件

1. 已构建：`./scripts/build.sh`
2. 摄像头可用：`./build/bin/webrtc_push_demo --list-cameras` 能看到设备

---

## 步骤 1：确保路由存在（必须）

在**任意终端**执行：

```bash
# 查看本机 IP
ip -4 route get 1 | awk '{print $7; exit}'

# 若输出为 192.168.3.222，则添加路由：
sudo ip route add 192.168.3.222/32 dev lo

# 若提示 "File exists"，说明路由已存在，跳过即可

# 确认路由
ip route show | grep "dev lo"
# 应看到: 192.168.3.222 dev lo scope link
```

---

## 步骤 2：启动推流（终端 1）

```bash
cd ~/PPS/wuba/webrtc_demo
export LD_LIBRARY_PATH="3rdparty/libwebrtc/lib/linux/arm64:$LD_LIBRARY_PATH"  # x64 改为 x64

./scripts/push.sh livestream /dev/video11
```

**等待直到看到：**
```
[PushStreamer] ICE 收集状态: Complete
```

**不要按 Ctrl+C**，保持推流运行。

---

## 步骤 3：启动拉流（终端 2）

**新开一个终端**，执行：

```bash
cd ~/PPS/wuba/webrtc_demo
export LD_LIBRARY_PATH="3rdparty/libwebrtc/lib/linux/arm64:$LD_LIBRARY_PATH"  # x64 改为 x64

# 本地拉流（SDL2 窗口播放）
./build/bin/webrtc_pull_demo 127.0.0.1:8765
```

**依赖**：需安装 SDL2，`sudo apt install libsdl2-dev`

**预期**：弹出 SDL2 窗口显示视频，按 Esc 或关闭窗口退出。

**跨机拉流**：在另一台 Linux 主机上，将 `127.0.0.1` 改为推流端 IP：
```bash
./build/bin/webrtc_pull_demo 192.168.3.222:8765
```
需确保信令服务器监听 0.0.0.0（`scripts/push.sh` 会启动），且防火墙放行 8765 端口。

---

## 步骤 4：停止

- 在**拉流终端**按 Ctrl+C 停止拉流
- 在**推流终端**按 Ctrl+C 停止推流

---

## 若仍拉不到帧

1. **确认路由**：`ip route show | grep "dev lo"` 必须有 `192.168.3.222`
2. **确认推流先就绪**：拉流必须在推流出现 `ICE 收集状态: Complete` 之后启动
3. **运行诊断**：`./scripts/diagnose_udp.sh`，查看完整输出

---

## 一键测试（单终端）

若希望一条命令完成测试：

```bash
cd ~/PPS/wuba/webrtc_demo
./scripts/diagnose_udp.sh
```

该脚本会自动完成：检查路由 → 启动信令 → 启动推流 → 启动拉流 → 输出结果。

**重要：从「3. 启动拉流」开始，请勿按 Ctrl+C，必须等待约 25 秒让脚本自动结束。** 若提前中断，拉流端可能尚未收到帧。
